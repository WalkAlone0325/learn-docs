## 1. 项目概述

### 1.1 项目架构

本演示项目是一个博客 API 系统，采用经典的三层架构。所有服务通过 Docker Compose 编排运行：

```
┌───────────────────────────────────────────────────────┐
│                      客户端                             │
│                (curl / Postman / 浏览器)                  │
└──────────────────────┬────────────────────────────────┘
                       │ HTTP
                       ▼
┌───────────────────────────────────────────────────────┐
│                  FastAPI 应用 (Uvicorn)                  │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐       │
│  │   API 路由   │  │   核心层    │  │   数据模型   │       │
│  │  (api/*.py) │  │ (security  │  │ (models/*) │       │
│  │             │  │  di)       │  │            │       │
│  └────────────┘  └────────────┘  └────────────┘       │
└──────────────────────┬────────────────────────────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
┌──────────────┐ ┌──────────┐ ┌──────────────┐
│  PostgreSQL   │ │   Redis   │ │    卷存储     │
│  (持久数据)   │ │ (缓存/限流)│ │  (pg 数据)   │
└──────────────┘ └──────────┘ └──────────────┘
```

Docker 网络层面，三个容器通过自定义网络 `blog-network` 通信：

```
┌─ blog-network ───────────────────────────────────┐
│  ┌──────────┐     ┌──────────┐     ┌──────────┐  │
│  │  app      │────▶│  db       │     │  redis   │  │
│  │ :8000    │     │ :5432    │     │ :6379    │  │
│  └──────────┘     └──────────┘     └──────────┘  │
│       │                                           │
└───────┼───────────────────────────────────────────┘
        │ :8000 (exposed)
        ▼
    宿主机 → 客户端
```

## 3. 完整代码实现

### 3.1 包文件与配置层

#### 3.1.1 `app/__init__.py`

**作用**: 将 `app/` 标记为 Python 包。

**为什么这样写**: Python 需要 `__init__.py` 才能将目录识别为包（Python 3.3+ 的 implicit namespace packages 虽然支持空包，但显式创建是最佳实践）。

```python
"""
blog-api application package.
"""
```

**关键点**: 即使是空文件，也要确保它存在，否则 Python 无法导入 `app` 下的模块。

---

#### 3.1.2 `app/config.py`

**作用**: 集中管理所有配置项，支持环境变量注入。

**教师视角**: 新手常犯的错误是将数据库 URL 等敏感信息硬编码在代码中。正确做法是使用环境变量，通过 Pydantic Settings 自动加载 `.env` 文件。

**工程师视角**: 使用 `@lru_cache` 装饰 `get_settings` 确保 Settings 单例，避免每次调用都重新解析环境变量。

```python
from pydantic_settings import BaseSettings


class Settings(BaseSettings):
    DATABASE_URL: str = "postgresql+asyncpg://postgres:postgres@localhost:5432/blog"
    REDIS_URL: str = "redis://localhost:6379/0"
    SECRET_KEY: str = "change-me-to-a-random-secret"
    ALGORITHM: str = "HS256"
    ACCESS_TOKEN_EXPIRE_MINUTES: int = 30
    APP_TITLE: str = "Blog API"
    APP_VERSION: str = "0.1.0"
    CORS_ORIGINS: str = "http://localhost:3000,http://localhost:8000"
    POOL_SIZE: int = 20
    MAX_OVERFLOW: int = 10
    POOL_RECYCLE: int = 3600
    POOL_PRE_PING: bool = True
    ECHO_SQL: bool = False

    model_config = {"env_file": ".env", "env_file_encoding": "utf-8", "extra": "ignore"}


from functools import lru_cache


@lru_cache
def get_settings() -> Settings:
    return Settings()
```

**关键点**:
- `model_config` 中的 `"extra": "ignore"` 允许 `.env` 中有额外变量时不报错
- `@lru_cache` 保证 Settings 是单例，减少 I/O 开销
- `.env` 文件中的变量优先级高于默认值
- 生产环境通过 Docker 环境变量注入，无需 `.env` 文件

💡 **提示**: Pydantic v2 使用 `model_config` 字典替代 v1 的 `class Config`。注意不要混用。

---

#### 3.1.3 `app/database.py`

**作用**: 创建异步引擎、会话工厂和声明式基类。

**教师视角**: AsyncSession 的生命周期管理是 SQLAlchemy 异步编程的核心。每个请求应当创建新的会话，请求结束后关闭。使用 `async_sessionmaker` 作为工厂，配合 FastAPI 的 `Depends` 实现自动管理。

**工程师视角**: 连接池参数直接影响数据库性能：
- `pool_size=20`: 池中保持 20 个连接
- `max_overflow=10`: 高峰期允许额外 10 个连接
- `pool_pre_ping=True`: 每次从池中取连接前检查可用性（防止使用断开的连接）
- `pool_recycle=3600`: 连接使用 1 小时后回收（防止长时间空闲连接被数据库端断开）

```python
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession
from sqlalchemy.orm import DeclarativeBase

from app.config import get_settings

settings = get_settings()

engine = create_async_engine(
    settings.DATABASE_URL,
    pool_size=settings.POOL_SIZE,
    max_overflow=settings.MAX_OVERFLOW,
    pool_pre_ping=settings.POOL_PRE_PING,
    pool_recycle=settings.POOL_RECYCLE,
    echo=settings.ECHO_SQL,
)

async_session = async_sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False,
)


class Base(DeclarativeBase):
    pass


async def get_db():
    async with async_session() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise
        finally:
            await session.close()
```

**关键点**:
- `expire_on_commit=False`: 防止 commit 后模型实例过期，导致后续访问属性时触发 lazy load
- `get_db` 使用 `async with` 确保会话自动关闭
- `yield` 配合 FastAPI 的 `Depends` 实现每个请求一个会话
- commit/rollback 在依赖中集中处理，路由代码更简洁

💡 **提示**: `expire_on_commit=False` 在异步模式下尤为重要。如果为 `True`，commit 后所有属性变为过期状态，访问时会尝试异步 lazy load，但此时 session 可能已被关闭。

#### 3.2.2 `app/models/user.py`

**作用**: 用户数据模型，存储认证信息和基本资料。

**教师视角**: 数据库模型关注的是"持久化存储"，与 API 请求/响应 schema 职责分离。密码字段存储的是哈希值而非明文。

**工程师视角**: `default` 参数传入的是函数引用（不加括号），每次创建新记录时都会执行该函数获取当前时间。`onupdate` 在每次更新时自动刷新时间戳。

```python
from datetime import datetime, timezone

from sqlalchemy import Column, Integer, String, DateTime
from sqlalchemy.orm import relationship

from app.database import Base


def _utcnow() -> datetime:
    return datetime.now(timezone.utc)


class User(Base):
    __tablename__ = "users"

    id = Column(Integer, primary_key=True, index=True)
    username = Column(String(50), unique=True, index=True, nullable=False)
    email = Column(String(100), unique=True, index=True, nullable=False)
    hashed_password = Column(String(255), nullable=False)
    created_at = Column(DateTime(timezone=True), default=_utcnow)
    updated_at = Column(DateTime(timezone=True), default=_utcnow, onupdate=_utcnow)

    posts = relationship("Post", back_populates="author", cascade="all, delete-orphan")
    comments = relationship("Comment", back_populates="author", cascade="all, delete-orphan")
```

**关键点**:
- `index=True` 为 username 和 email 创建数据库索引，加速登录查询
- `unique=True` 确保用户名和邮箱唯一
- `_utcnow` 是函数引用（不加括号），ORM 在 INSERT 时调用
- `cascade="all, delete-orphan"` 在删除用户时自动删除所有关联的文章和评论
- `DateTime(timezone=True)` 存储带时区的时间戳，避免时区歧义

💡 **提示**: 如果 `default=_utcnow()`（加了括号），ORM 会在**定义模型时**执行一次函数，所有记录共享同一个时间戳。这是非常常见的错误。

#### 3.2.3 `app/models/post.py`

**作用**: 文章数据模型，包含标题、内容和作者关联。

**工程师视角**: `ForeignKey` 需要在数据库中创建外键约束，确保数据完整性。`relationship` 则在 ORM 层面建立对象关联。

```python
from datetime import datetime, timezone

from sqlalchemy import Column, Integer, String, Text, DateTime, ForeignKey
from sqlalchemy.orm import relationship

from app.database import Base


def _utcnow() -> datetime:
    return datetime.now(timezone.utc)


class Post(Base):
    __tablename__ = "posts"

    id = Column(Integer, primary_key=True, index=True)
    title = Column(String(200), nullable=False, index=True)
    content = Column(Text, nullable=False)
    user_id = Column(Integer, ForeignKey("users.id", ondelete="CASCADE"), nullable=False, index=True)
    created_at = Column(DateTime(timezone=True), default=_utcnow)
    updated_at = Column(DateTime(timezone=True), default=_utcnow, onupdate=_utcnow)

    author = relationship("User", back_populates="posts")
    comments = relationship("Comment", back_populates="post", cascade="all, delete-orphan")
```

**关键点**:
- `ForeignKey("users.id", ondelete="CASCADE")`: 用户被删除时，该用户的所有文章自动删除（数据库级）
- `cascade="all, delete-orphan"`: 删除文章时自动删除所有关联评论（ORM 级）
- 双重级联确保无论通过数据库还是 ORM 删除，关联数据都能被清理
- `Text` 类型适合存储长文本内容
- `ondelete="CASCADE"` 是数据库级别约束，与 ORM 的 cascade 互补

```python
from datetime import datetime, timezone
```

#### 3.3.2 `app/schemas/user.py`

**作用**: 定义用户的请求和响应数据结构。

**教师视角**: Schema 与 Model 职责分离——Model 关心数据库存储，Schema 关心 API 通信。密码字段只在创建请求中出现，永远不应出现在响应中。

**工程师视角**: Pydantic v2 使用 `ConfigDict(from_attributes=True)` 替代 v1 的 `class Config: orm_mode = True`。支持从 ORM 实例自动创建 Pydantic 对象。

```python
from datetime import datetime

from pydantic import BaseModel, ConfigDict, EmailStr


class UserCreate(BaseModel):
    username: str
    email: EmailStr
    password: str


class UserRead(BaseModel):
    model_config = ConfigDict(from_attributes=True)

    id: int
    username: str
    email: str
    created_at: datetime
    updated_at: datetime


class UserUpdate(BaseModel):
    username: str | None = None
    email: EmailStr | None = None
    password: str | None = None
```

**关键点**:
- `EmailStr` 需要安装 `email-validator` 包，自动验证邮箱格式
- `UserCreate` 包含 `password` 明文密码，用于注册输入
- `UserRead` **不**包含 `hashed_password` 或 `password`，确保密码永不泄漏
- `UserUpdate` 所有字段可选，支持部分更新
- `from_attributes=True` 允许直接从 SQLAlchemy 模型创建（用于 `response_model`）

#### 3.3.3 `app/schemas/post.py`

**作用**: 定义文章的请求/响应数据结构，包括分页响应。

```python
from datetime import datetime

from pydantic import BaseModel, ConfigDict

from app.schemas.user import UserRead


class PostCreate(BaseModel):
    title: str
    content: str


class PostUpdate(BaseModel):
    title: str | None = None
    content: str | None = None


class PostRead(BaseModel):
    model_config = ConfigDict(from_attributes=True)

    id: int
    title: str
    content: str
    user_id: int
    created_at: datetime
    updated_at: datetime
    author: UserRead | None = None


class PostListResponse(BaseModel):
    items: list[PostRead]
    total: int
    page: int
    page_size: int
    total_pages: int
```

**关键点**:
- `author` 字段嵌套了 `UserRead`，在 API 响应中自动包含作者信息
- `PostListResponse` 封装分页元数据，客户端无需额外计算
- `PostUpdate` 所有字段可选，支持 PATCH 语义
- 嵌套 schema 通过 SQLAlchemy 的 `joinedload` 预加载避免 N+1

#### 3.4.2 `app/core/security.py`

**作用**: 提供密码哈希、验证和 JWT 创建/验证功能。

**教师视角**: 密码绝不能以明文形式存储。使用 bcrypt 哈希算法，它内置了 salt（盐值）和 cost factor（计算成本因子），即使数据库泄漏也难以破解。

**工程师视角**:
- JWT 的 `exp` 声明在解码时由 `python-jose` 自动验证，过期令牌将被拒绝
- `sub` (subject) 通常是用户标识符（这里是 username）
- JWT payload 中不存储敏感信息（如密码），因为 JWT 只签名不加密
- `ACCESS_TOKEN_EXPIRE_MINUTES` 控制令牌有效期，生产环境建议 15-30 分钟

```python
from datetime import datetime, timedelta, timezone

from jose import jwt, JWTError
from passlib.context import CryptContext

from app.config import get_settings

settings = get_settings()

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")


def hash_password(password: str) -> str:
    return pwd_context.hash(password)


def verify_password(plain_password: str, hashed_password: str) -> bool:
    return pwd_context.verify(plain_password, hashed_password)


def create_access_token(data: dict, expires_delta: timedelta | None = None) -> str:
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + (
        expires_delta or timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES)
    )
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, settings.SECRET_KEY, algorithm=settings.ALGORITHM)


def decode_access_token(token: str) -> dict | None:
    try:
        payload = jwt.decode(token, settings.SECRET_KEY, algorithms=[settings.ALGORITHM])
        return payload
    except JWTError:
        return None
```

**关键点**:
- `CryptContext(schemes=["bcrypt"], deprecated="auto")` 使用 bcrypt 算法，并自动处理哈希升级
- `datetime.now(timezone.utc)` 替代已废弃的 `datetime.utcnow()`（Python 3.12+ 推荐）
- JWT 的 `exp` 使用 UTC 时间，避免时区问题
- `decode_access_token` 返回 `dict | None`，调用方根据 `None` 判断认证失败
- `deprecated="auto"` 允许在新用户使用更强算法时自动迁移旧哈希

#### 3.4.3 `app/core/dependencies.py`

**作用**: 定义 FastAPI 依赖注入函数，提供会话管理、用户认证和分页参数。

**教师视角**: 依赖注入是 FastAPI 的核心优势。它将"获取当前用户"这样的横切关注点与路由处理逻辑分离，使代码更简洁、可测试。

**工程师视角**: `HTTPBearer` 自动从 `Authorization: Bearer <token>` 头中提取令牌。`Depends(get_current_user)` 可以保护任何需要认证的路由。

```python
from fastapi import Depends, HTTPException, Query, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.core.security import decode_access_token
from app.models.user import User

security_scheme = HTTPBearer()


async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security_scheme),
    db: AsyncSession = Depends(get_db),
) -> User:
    payload = decode_access_token(credentials.credentials)
    if payload is None:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="无效的访问令牌",
            headers={"WWW-Authenticate": "Bearer"},
        )

    username: str | None = payload.get("sub")
    if username is None:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="令牌中缺少用户标识",
            headers={"WWW-Authenticate": "Bearer"},
        )

    result = await db.execute(select(User).where(User.username == username))
    user = result.scalar_one_or_none()
    if user is None:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="用户不存在",
            headers={"WWW-Authenticate": "Bearer"},
        )

    return user


async def pagination_params(
    page: int = Query(1, ge=1, description="页码"),
    page_size: int = Query(20, ge=1, le=100, description="每页数量"),
) -> tuple[int, int]:
    return page, page_size
```

**关键点**:
- `HTTPBearer` 自动处理 `Authorization` 头解析
- `Depends(get_current_user)` 可用作路由保护装饰器的参数
- `headers={"WWW-Authenticate": "Bearer"}` 让客户端知道应该使用 Bearer 认证
- `pagination_params` 返回元组，路由中可解包为 `(page, page_size)`
- `scalar_one_or_none()` 返回单个结果或 `None`（如有多条则报错）

💡 **提示**: `WWW-Authenticate` 响应头是 HTTP 标准的一部分，帮助客户端正确处理认证流程。

#### 3.5.2 `app/api/auth.py`

**作用**: 处理用户注册和登录，是用户进入系统的入口。

**教师视角**: 注册时需要验证用户名和邮箱的唯一性。不要一次性加载所有用户然后在 Python 中比较——利用数据库的唯一索引和 `select` 查询，让数据库做它擅长的事。

**工程师视角**: 密码在存储前必须哈希。注册时使用 `hash_password`，登录时使用 `verify_password`。永远不要记录或返回密码。

```python
from fastapi import APIRouter, Depends, HTTPException, status
from sqlalchemy import select, or_
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.schemas.user import UserCreate, UserRead
from app.schemas.auth import LoginRequest, TokenResponse
from app.core.security import hash_password, verify_password, create_access_token
from app.models.user import User

router = APIRouter(prefix="/api/auth", tags=["auth"])


@router.post("/register", response_model=UserRead, status_code=status.HTTP_201_CREATED)
async def register(user_data: UserCreate, db: AsyncSession = Depends(get_db)):
    existing = await db.execute(
        select(User).where(
            or_(User.username == user_data.username, User.email == user_data.email)
        )
    )
    if existing.scalar_one_or_none():
        raise HTTPException(
            status_code=status.HTTP_409_CONFLICT,
            detail="用户名或邮箱已存在",
        )

    new_user = User(
        username=user_data.username,
        email=user_data.email,
        hashed_password=hash_password(user_data.password),
    )
    db.add(new_user)
    await db.commit()
    await db.refresh(new_user)
    return new_user


@router.post("/login", response_model=TokenResponse)
async def login(login_data: LoginRequest, db: AsyncSession = Depends(get_db)):
    result = await db.execute(
        select(User).where(User.username == login_data.username)
    )
    user = result.scalar_one_or_none()

    if not user or not verify_password(login_data.password, user.hashed_password):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="用户名或密码错误",
        )

    access_token = create_access_token(data={"sub": user.username})
    return TokenResponse(access_token=access_token)
```

**关键点**:
- 注册使用 `or_` 一次查询同时检查用户名和邮箱唯一性
- `db.refresh(new_user)` 从数据库重新加载以获取生成的 id 和时间戳
- 登录使用明确的错误信息"用户名或密码错误"，不暴露是哪个信息错误（安全考量）
- `status_code=status.HTTP_201_CREATED` 符合 RESTful 语义
- 密码永远不会出现在响应模型中（UserRead 中没有 password/hashed_password 字段）

#### 3.5.3 `app/api/users.py`

**作用**: 用户资料管理，获取和更新个人信息。

**教师视角**: 资源所有权是 API 安全的重要概念。用户只能查看和更新自己的信息。管理员功能（查看所有用户）可后续扩展。

**工程师视角**: 更新时只提供非空字段，使用 `UserUpdate` schema 的 `model_dump(exclude_unset=True)` 仅更新客户端提供的字段。

```python
from fastapi import APIRouter, Depends, HTTPException, status
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.schemas.user import UserRead, UserUpdate
from app.core.security import hash_password
from app.core.dependencies import get_current_user
from app.models.user import User

router = APIRouter(prefix="/api/users", tags=["users"])


@router.get("/me", response_model=UserRead)
async def get_current_user_info(current_user: User = Depends(get_current_user)):
    return current_user


@router.put("/me", response_model=UserRead)
async def update_current_user(
    user_data: UserUpdate,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    update_fields = user_data.model_dump(exclude_unset=True)

    if not update_fields:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="没有提供需要更新的字段",
        )

    if "username" in update_fields:
        existing = await db.execute(
            select(User).where(
                User.username == update_fields["username"],
                User.id != current_user.id,
            )
        )
        if existing.scalar_one_or_none():
            raise HTTPException(
                status_code=status.HTTP_409_CONFLICT,
                detail="用户名已存在",
            )
        current_user.username = update_fields["username"]

    if "email" in update_fields:
        existing = await db.execute(
            select(User).where(
                User.email == update_fields["email"],
                User.id != current_user.id,
            )
        )
        if existing.scalar_one_or_none():
            raise HTTPException(
                status_code=status.HTTP_409_CONFLICT,
                detail="邮箱已被使用",
            )
        current_user.email = update_fields["email"]

    if "password" in update_fields:
        current_user.hashed_password = hash_password(update_fields["password"])

    await db.commit()
    await db.refresh(current_user)
    return current_user


@router.get("/{user_id}", response_model=UserRead)
async def get_user(
    user_id: int,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    result = await db.execute(select(User).where(User.id == user_id))
    user = result.scalar_one_or_none()
    if not user:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="用户不存在",
        )
    return user
```

**关键点**:
- `/me` 端点使用 `Depends(get_current_user)` 直接返回当前用户
- 更新用户名/邮箱时需排除当前用户检查唯一性
- `model_dump(exclude_unset=True)` 只包含客户端发送的字段，支持部分更新
- 用户只能通过 `/me` 端点更新自己的信息

#### 3.5.4 `app/api/posts.py`

**作用**: 文章的增删改查，包含分页、搜索、排序功能。

**教师视角**: 分页是列表接口的基本要求。使用 `OFFSET/LIMIT` 是常见的分页方式（适用于中小数据集）。搜索使用 `ILIKE` 实现大小写不敏感的模糊匹配。

**工程师视角**: `joinedload(Post.author)` 使用 SQL JOIN 一次性加载作者信息，避免 N+1 查询问题。每个文章的评论不会被加载（直到需要时）。

```python
from fastapi import APIRouter, Depends, HTTPException, Query, status
from sqlalchemy import select, func, asc as asc_func, desc as desc_func
from sqlalchemy.orm import joinedload
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.schemas.post import PostCreate, PostRead, PostUpdate, PostListResponse
from app.models.post import Post
from app.models.user import User
from app.core.dependencies import get_current_user, pagination_params

router = APIRouter(prefix="/api/posts", tags=["posts"])

SORTABLE_FIELDS = {"created_at", "updated_at", "title", "id"}


@router.post("", response_model=PostRead, status_code=status.HTTP_201_CREATED)
async def create_post(
    post_data: PostCreate,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    new_post = Post(
        title=post_data.title,
        content=post_data.content,
        user_id=current_user.id,
    )
    db.add(new_post)
    await db.commit()
    await db.refresh(new_post)

    result = await db.execute(
        select(Post).options(joinedload(Post.author)).where(Post.id == new_post.id)
    )
    return result.unique().scalar_one()


@router.get("", response_model=PostListResponse)
async def list_posts(
    search: str | None = Query(None, description="标题搜索关键词"),
    sort: str = Query("created_at", description="排序字段"),
    order: str = Query("desc", description="排序方向"),
    db: AsyncSession = Depends(get_db),
    pagination: tuple[int, int] = Depends(pagination_params),
):
    page, page_size = pagination

    if sort not in SORTABLE_FIELDS:
        sort = "created_at"

    order_fn = desc_func if order == "desc" else asc_func
    base_query = select(Post).options(joinedload(Post.author))

    count_query = select(func.count()).select_from(Post)

    if search:
        search_filter = Post.title.ilike(f"%{search}%")
        base_query = base_query.where(search_filter)
        count_query = count_query.where(search_filter)

    total_result = await db.execute(count_query)
    total = total_result.scalar_one()

    sort_column = getattr(Post, sort, Post.created_at)
    base_query = base_query.order_by(order_fn(sort_column))
    base_query = base_query.offset((page - 1) * page_size).limit(page_size)

    result = await db.execute(base_query)
    posts = result.unique().scalars().all()

    return PostListResponse(
        items=list(posts),
        total=total,
        page=page,
        page_size=page_size,
        total_pages=(total + page_size - 1) // page_size,
    )


@router.get("/{post_id}", response_model=PostRead)
async def get_post(post_id: int, db: AsyncSession = Depends(get_db)):
    result = await db.execute(
        select(Post).options(joinedload(Post.author)).where(Post.id == post_id)
    )
    post = result.unique().scalar_one_or_none()
    if not post:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="文章不存在",
        )
    return post


@router.put("/{post_id}", response_model=PostRead)
async def update_post(
    post_id: int,
    post_data: PostUpdate,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    result = await db.execute(
        select(Post).options(joinedload(Post.author)).where(Post.id == post_id)
    )
    post = result.unique().scalar_one_or_none()
    if not post:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="文章不存在",
        )

    if post.user_id != current_user.id:
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail="无权修改此文章",
        )

    update_fields = post_data.model_dump(exclude_unset=True)
    for field, value in update_fields.items():
        setattr(post, field, value)

    await db.commit()
    await db.refresh(post)
    return post


@router.delete("/{post_id}", status_code=status.HTTP_204_NO_CONTENT)
async def delete_post(
    post_id: int,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    result = await db.execute(select(Post).where(Post.id == post_id))
    post = result.scalar_one_or_none()
    if not post:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="文章不存在",
        )

    if post.user_id != current_user.id:
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail="无权删除此文章",
        )

    await db.delete(post)
    await db.commit()
```

**关键点**:
- `SORTABLE_FIELDS` 白名单防止 SQL 注入（用户不能通过 sort 参数注入任意 SQL）
- `result.unique().scalars().all()` 在 `joinedload` 后必须调用 `unique()` 去重（因 JOIN 可能产生重复行）
- `204 No Content` 是删除操作的 RESTful 响应
- `403 Forbidden` 用于资源所有权检查
- 搜索使用 `ILIKE` 进行大小写不敏感的模糊匹配

💡 **提示**: `joinedload` 是**仅用于读取**的加载策略。不要在 `joinedload` 加载的对象上修改关联数据并期望自动持久化——这种情况下应使用 `select` 或 `subqueryload`。

#### 3.5.5 `app/api/comments.py`

**作用**: 文章的评论管理，支持添加、列表和删除。

**教师视角**: 评论是嵌套资源（属于文章），API 路径设计体现了这种层级关系：`/api/posts/{post_id}/comments`。

**工程师视角**: 添加评论时验证文章存在性，删除评论时检查资源所有权。级联删除由数据库和 ORM 共同保证。

```python
from fastapi import APIRouter, Depends, HTTPException, status
from sqlalchemy import select, func
from sqlalchemy.orm import joinedload
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.schemas.comment import CommentCreate, CommentRead
from app.models.comment import Comment
from app.models.post import Post
from app.models.user import User
from app.core.dependencies import get_current_user

@router.post("/api/posts/{post_id}/comments", response_model=CommentRead, status_code=status.HTTP_201_CREATED)
async def create_comment(

    db.add(new_comment)
    await db.commit()
    await db.refresh(new_comment)

    result = await db.execute(

@router.get("/api/posts/{post_id}/comments", response_model=list[CommentRead])
async def list_comments(post_id: int, db: AsyncSession = Depends(get_db)):

        .order_by(Comment.created_at.desc())

    db: AsyncSession = Depends(get_db),

):
    result = await db.execute(
        select(Comment).options(joinedload(Comment.post)).where(Comment.id == comment_id)

    comment = result.unique().scalar_one_or_none()
    if not comment:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="评论不存在",
        )

    if comment.user_id != current_user.id:
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,

        )

```

**关键点**:
- 评论路由前缀不统一——`create_comment` 和 `list_comments` 路径以 `/api/posts/{post_id}/comments` 为前缀，`delete_comment` 以 `/api/comments/{comment_id}` 为前缀

### 3.6 应用入口

#### 3.6.1 `app/main.py`

**作用**: FastAPI 应用创建、生命周期管理、中间件注册和路由挂载。

**教师视角**: FastAPI 的 `lifespan` 参数取代了已废弃的 `on_event("startup")` 和 `on_event("shutdown")`。异步上下文管理器确保启动和关闭逻辑成对出现。

**工程师视角**: CORS 中间件的 `allow_origins` 在生产环境应限制为具体的前端域名，不要使用 `["*"]`。`allow_credentials=True` 时不能使用 `"*"`。

```python
from contextlib import asynccontextmanager

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

from app.config import get_settings
from app.database import engine, Base
from app.api import auth_router, users_router, posts_router, comments_router

settings = get_settings()


@asynccontextmanager
async def lifespan(app: FastAPI):
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    yield
    await engine.dispose()


app = FastAPI(
    title=settings.APP_TITLE,
    version=settings.APP_VERSION,
    lifespan=lifespan,
)

origins = [origin.strip() for origin in settings.CORS_ORIGINS.split(",")]

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

app.include_router(auth_router)
app.include_router(users_router)
app.include_router(posts_router)
app.include_router(comments_router)


@app.get("/health", tags=["health"])
async def health_check():
    return {"status": "ok", "version": settings.APP_VERSION}
```

**关键点**:
- `lifespan` 启动时自动创建所有表（开发环境方便，生产环境应使用 Alembic）
- `engine.dispose()` 在关闭时释放所有数据库连接
- CORS origins 从配置中读取，支持逗号分隔的多个域名
- `/health` 健康检查端点在 Docker 环境中可用于容器健康检查
- `Base.metadata.create_all` 在开发阶段快速建表，生产环境应禁用此自动化

💡 **提示**: `lifespan` 是 Python 3.7+ 的 `@asynccontextmanager`，FastAPI 0.93+ 开始推荐使用。不要将 `lifespan` 与 `on_event` 混用。

### 3.7 Docker 配置

#### 3.7.1 `Dockerfile`

**作用**: 使用多阶段构建创建最小化生产镜像。

**教师视角**: 多阶段构建是 Docker 最佳实践。第一阶段 (builder) 安装编译依赖和 Python 包，第二阶段 (production) 只复制编译后的包和运行时代码，大幅减小镜像体积。

**工程师视角**: 使用 `uv`（Rust 编写的 Python 包管理器）可以比 `pip` 快 10-100 倍。`--no-cache-dir` 避免缓存占用空间。`.dockerignore` 防止将本地无关文件复制到镜像。

```dockerfile
# Stage 1: Builder - 编译依赖
FROM python:3.14-slim AS builder

WORKDIR /app

RUN pip install --no-cache-dir uv

COPY pyproject.toml requirements.txt ./
RUN uv pip install --system --no-cache -r requirements.txt

# Stage 2: Production - 运行环境
FROM python:3.14-slim AS production

WORKDIR /app

COPY --from=builder /usr/local/lib/python3.14/site-packages /usr/local/lib/python3.14/site-packages
COPY . .

EXPOSE 8000

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**关键点**:
- 第一阶段使用 `uv` 加速依赖安装
- 第二阶段只复制编译后的 site-packages 和应用代码
- `COPY . .` 必须在最后（利用 Docker 缓存机制，代码变更时不会导致依赖重新安装）
- `python:3.14-slim` 比完整版小约 50%，基于 Debian slim
- `EXPOSE 8000` 是文档性质，不影响实际端口映射

💡 **提示**: Docker 镜像层的缓存机制——每一行 `RUN`/`COPY` 指令都会创建新层。仅当该行内容变化时才会重新构建该层及后续层。因此应将不常变化的指令放在前面（如安装依赖），经常变化的指令放在后面（如复制代码）。

#### 3.7.2 `docker-compose.yml`

**作用**: 定义和运行多容器 Docker 应用，编排 app、postgres 和 redis 三个服务。

**教师视角**: Docker Compose 将"如何启动整个系统"声明为 YAML 配置，实现环境的一致性和可重复性。

**工程师视角**: 生产级配置需要考虑：健康检查（Healthcheck）、持久化卷、网络隔离、日志限制、重启策略。

```yaml
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      DATABASE_URL: "postgresql+asyncpg://postgres:postgres@db:5432/blog"
      REDIS_URL: "redis://redis:6379/0"
      SECRET_KEY: "${SECRET_KEY:-change-me-in-production}"
      CORS_ORIGINS: "${CORS_ORIGINS:-http://localhost:3000}"
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
    restart: unless-stopped
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: blog
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres -d blog"]
      interval: 5s
      timeout: 5s
      retries: 5
      start_period: 10s
    restart: unless-stopped
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5
    restart: unless-stopped
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

volumes:
  postgres_data:
    driver: local
  redis_data:
    driver: local
```

**关键点**:
- `depends_on` 只保证启动顺序，不保证服务就绪。必须配合 `condition: service_healthy` 等待服务真正可用
- `healthcheck` 让 Docker 知道服务是否已就绪，`pg_isready` 检查 PostgreSQL
- 命名卷 `postgres_data` 和 `redis_data` 持久化数据，容器重建后数据不丢失
- 日志限制避免日志文件无限增长占用磁盘
- `${SECRET_KEY:-change-me-in-production}` 支持环境变量覆盖，`:-` 提供默认值

💡 **提示**: Docker Compose v2.40+ 不再要求 `version: "3"` 字段（新版 Compose 自动推断）。`depends_on` 的 `condition` 字段需要 Compose 2.20+ 支持。

#### 3.9.2 `alembic/env.py`

**作用**: Alembic 运行时环境配置，导入所有模型以便自动检测 schema 变更。

**教师视角**: 这是最常见的"问题文件"——如果忘记导入模型，Alembic 无法知道表结构，生成的迁移会是空的。

**工程师视角**: 异步数据库需要使用 `async_engine_from_config` 和 `run_sync` 来运行迁移。

```python
import asyncio
from logging.config import fileConfig

from sqlalchemy import pool
from sqlalchemy.ext.asyncio import async_engine_from_config

from alembic import context

config = context.config
fileConfig(config.config_file_name)

from app.database import Base
from app.models import User, Post, Comment

target_metadata = Base.metadata


def run_migrations_offline():
    url = config.get_main_option("sqlalchemy.url")
    context.configure(
        url=url,
        target_metadata=target_metadata,
        literal_binds=True,
        dialect_opts={"paramstyle": "named"},
    )
    with context.begin_transaction():
        context.run_migrations()


def do_run_migrations(connection):
    context.configure(connection=connection, target_metadata=target_metadata)
    with context.begin_transaction():
        context.run_migrations()


async def run_async_migrations():
    connectable = async_engine_from_config(
        config.get_section(config.config_ini_section, {}),
        prefix="sqlalchemy.",
        poolclass=pool.NullPool,
    )
    async with connectable.connect() as connection:
        await connection.run_sync(do_run_migrations)
    await connectable.dispose()


def run_migrations_online():
    asyncio.run(run_async_migrations())


if context.is_offline_mode():
    run_migrations_offline()
else:
    run_migrations_online()
```

**关键点**:
- 导入 `User`, `Post`, `Comment` 是必要的——没有这行，Alembic 不知道这些表的存在
- `async_engine_from_config` 创建异步引擎用于迁移
- `poolclass=pool.NullPool` 避免迁移时使用连接池（每次迁移只使用一个连接）
- `run_migrations_online` 使用 `asyncio.run()` 同步调用异步迁移函数

### 5.1 软删除（Soft Delete）

**问题**: 物理删除数据不可恢复，可能导致用户误操作后数据永久丢失。软删除通过标记（而非删除）数据来保留恢复能力。

**实现**: 在模型中添加 `deleted_at` 字段，查询时默认过滤已删除的记录。

**代码示例:**

```python
# soft_delete_mixin.py
from datetime import datetime, timezone

from sqlalchemy import Column, DateTime
from sqlalchemy import select as sa_select


def _utcnow() -> datetime:
    return datetime.now(timezone.utc)


class SoftDeleteMixin:
    deleted_at = Column(DateTime(timezone=True), nullable=True, default=None)


def apply_soft_delete_filter(query, model):
    return query.where(model.deleted_at.is_(None))


async def soft_delete(db, instance):
    instance.deleted_at = datetime.now(timezone.utc)
    await db.commit()


async def restore(db, instance):
    instance.deleted_at = None
    await db.commit()


# 使用示例 - 修改 Post 模型
# 在 app/models/post.py 中:
# class Post(Base, SoftDeleteMixin):
#     ...

# 在查询中使用:
# query = apply_soft_delete_filter(select(Post), Post)
# result = await db.execute(query)


# 完整示例 - 软删除路由器
from fastapi import APIRouter, Depends, HTTPException, status

from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db, Base
from app.models.post import Post
from app.models.user import User
from app.core.dependencies import get_current_user

soft_delete_router = APIRouter(prefix="/api/posts", tags=["soft-delete"])


@soft_delete_router.delete("/{post_id}/soft", status_code=status.HTTP_200_OK)
async def soft_delete_post(
    post_id: int,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    result = await db.execute(
        select(Post).where(Post.id == post_id, Post.deleted_at.is_(None))
    )
    post = result.scalar_one_or_none()
    if not post:
        raise HTTPException(status_code=404, detail="文章不存在")

    if post.user_id != current_user.id:
        raise HTTPException(status_code=403, detail="无权操作")

    await soft_delete(db, post)
    return {"message": "已软删除", "id": post_id}


@soft_delete_router.post("/{post_id}/restore", status_code=status.HTTP_200_OK)
async def restore_post(
    post_id: int,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    result = await db.execute(
        select(Post).where(Post.id == post_id, Post.deleted_at.is_not(None))
    )
    post = result.scalar_one_or_none()
    if not post:
        raise HTTPException(status_code=404, detail="文章不存在或未被删除")

    if post.user_id != current_user.id:
        raise HTTPException(status_code=403, detail="无权操作")

    await restore(db, post)
    return {"message": "已恢复", "id": post_id}
```

**关键点**:
- `deleted_at` 为 `NULL` 表示未删除，有值时表示删除时间
- 所有查询默认必须加 `deleted_at.is_(None)` 过滤
- 软删除与唯一索引冲突——如需用户名可重用，应使用复合唯一索引包含 `deleted_at`

### 5.2 批量操作（Batch Operations）

**问题**: 逐一创建/删除大量数据会产生大量数据库交互，性能低下。批量操作将多次交互合并为一次。

**代码示例:**

```python
# batch_operations.py
from fastapi import APIRouter, Depends, HTTPException, status
from pydantic import BaseModel, ConfigDict
from sqlalchemy import select, delete as sa_delete
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.models.post import Post
from app.models.user import User
from app.core.dependencies import get_current_user

batch_router = APIRouter(prefix="/api/batch", tags=["batch"])


class BatchCreateItem(BaseModel):
    title: str
    content: str


class BatchCreateRequest(BaseModel):
    items: list[BatchCreateItem]


class BatchCreateResponse(BaseModel):
    model_config = ConfigDict(from_attributes=True)
    created: int
    ids: list[int]


@batch_router.post("/posts", response_model=BatchCreateResponse, status_code=201)
async def batch_create_posts(
    request: BatchCreateRequest,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    if len(request.items) > 100:
        raise HTTPException(status_code=400, detail="单次批量创建不可超过 100 条")

    posts = [
        Post(title=item.title, content=item.content, user_id=current_user.id)
        for item in request.items
    ]
    db.add_all(posts)
    await db.commit()

    ids = [post.id for post in posts]
    return BatchCreateResponse(created=len(posts), ids=ids)


class BatchDeleteRequest(BaseModel):
    ids: list[int]


@batch_router.post("/posts/delete", status_code=200)
async def batch_delete_posts(
    request: BatchDeleteRequest,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    if len(request.ids) > 100:
        raise HTTPException(status_code=400, detail="单次批量删除不可超过 100 条")

    result = await db.execute(
        select(Post).where(
            Post.id.in_(request.ids),
            Post.user_id == current_user.id,
        )
    )
    posts = result.scalars().all()

    if len(posts) != len(request.ids):
        not_found = set(request.ids) - {p.id for p in posts}
        raise HTTPException(
            status_code=404,
            detail=f"以下文章不存在或无权删除: {not_found}",
        )

    for post in posts:
        await db.delete(post)
    await db.commit()

    return {"message": f"已删除 {len(posts)} 篇文章", "deleted": len(posts)}
```

**curl 示例:**

```bash
# 批量创建
curl -X POST http://localhost:8000/api/batch/posts \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "items": [
      {"title": "文章1", "content": "内容1"},
      {"title": "文章2", "content": "内容2"},
      {"title": "文章3", "content": "内容3"}
    ]
  }'

# 批量删除
curl -X POST http://localhost:8000/api/batch/posts/delete \

```

**关键点**:
- 批量操作始终有上限（此处 100 条），避免请求体过大或数据库锁表
- `db.add_all(posts)` 批量添加多个对象，效率高于循环 `db.add()`
- 批量删除先查询所有权再删除，确保安全

### 5.3 CSV 流式导出（Streaming Export）

**问题**: 导出大量数据时，全部加载到内存再返回会导致 OOM。流式响应边生成边发送，内存开销恒定。

**代码示例:**

```python
# csv_export.py
import csv
import io
from datetime import datetime, timezone

from fastapi import APIRouter, Depends
from fastapi.responses import StreamingResponse
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.models.post import Post
from app.models.user import User
from app.core.dependencies import get_current_user

csv_router = APIRouter(prefix="/api/export", tags=["export"])


async def generate_posts_csv(db: AsyncSession, user_id: int | None = None):
    query = select(Post).order_by(Post.created_at)
    if user_id:
        query = query.where(Post.user_id == user_id)

    result = await db.stream(query)
    header = ["ID", "标题", "内容摘要", "用户ID", "创建时间", "更新时间"]
    yield ",".join(header) + "\n"

    async for row in result:
        post = row[0]
        content_preview = post.content[:50].replace('"', '""').replace("\n", " ")
        line = [
            str(post.id),
            f'"{post.title}"',
            f'"{content_preview}"',
            str(post.user_id),
            post.created_at.isoformat() if post.created_at else "",
            post.updated_at.isoformat() if post.updated_at else "",
        ]
        yield ",".join(line) + "\n"


@csv_router.get("/posts/csv")
async def export_posts_csv(
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    return StreamingResponse(
        generate_posts_csv(db, user_id=current_user.id),
        media_type="text/csv",
        headers={
            "Content-Disposition": f"attachment; filename=posts_{datetime.now(timezone.utc).strftime('%Y%m%d')}.csv",
        },
    )
```

**curl 示例:**

```bash
curl http://localhost:8000/api/export/posts/csv \
  -H "Authorization: Bearer $TOKEN" \
  -o posts_export.csv
```

**关键点**:
- `StreamingResponse` 接受异步生成器，数据边生成边传输
- `db.stream()` 实现数据库端的游标式读取，避免一次性加载所有数据
- CSV 字段中如果包含逗号、引号或换行，需要正确转义
- `Content-Disposition: attachment` 强制浏览器下载文件

### 5.5 速率限制（Rate Limiting）

**问题**: 恶意用户可能频繁请求耗尽服务器资源。基于 Redis 的速率限制可以在分布式环境中共享限流状态。

**代码示例:**

```python
# rate_limiter.py
import time
from collections.abc import Callable
from functools import wraps

import redis.asyncio as aioredis
from fastapi import HTTPException, Request, status

from app.config import get_settings

settings = get_settings()

redis_client = aioredis.from_url(settings.REDIS_URL, decode_responses=True)


class RateLimiter:
    def __init__(self, max_requests: int, window_seconds: int, prefix: str = "rl"):
        self.max_requests = max_requests
        self.window_seconds = window_seconds
        self.prefix = prefix

    async def check(self, key: str) -> None:
        now = int(time.time())
        window_key = f"{self.prefix}:{key}:{now // self.window_seconds}"

        current = await redis_client.incr(window_key)
        if current == 1:
            await redis_client.expire(window_key, self.window_seconds)

        if current > self.max_requests:
            raise HTTPException(
                status_code=status.HTTP_429_TOO_MANY_REQUESTS,
                detail=f"请求过于频繁，请稍后再试（{self.max_requests} 次/{self.window_seconds} 秒）",
            )

    async def get_remaining(self, key: str) -> int:
        now = int(time.time())
        window_key = f"{self.prefix}:{key}:{now // self.window_seconds}"
        current = await redis_client.get(window_key)
        return max(0, self.max_requests - (int(current or 0)))


global_rate_limiter = RateLimiter(max_requests=60, window_seconds=60, prefix="rl:global")
auth_rate_limiter = RateLimiter(max_requests=5, window_seconds=60, prefix="rl:auth")


async def global_rate_limit(request: Request):
    client_ip = request.client.host if request.client else "unknown"
    await global_rate_limiter.check(client_ip)


async def auth_rate_limit(request: Request):
    client_ip = request.client.host if request.client else "unknown"
    await auth_rate_limiter.check(f"auth:{client_ip}")


# 使用示例 - 在 FastAPI 路由中
# @router.post("/login")
# async def login(..., _rate_limit=Depends(auth_rate_limit)):
#     ...

# 或全局中间件
# app.add_middleware(...)  # 此处略


# 测试速率限制
async def test_rate_limiter():
    limiter = RateLimiter(max_requests=3, window_seconds=10, prefix="rl:test")
    for i in range(5):
        try:
            await limiter.check("test_user")
            print(f"请求 {i+1}: 通过")
        except HTTPException as e:
            print(f"请求 {i+1}: 拒绝 - {e.detail}")
            break
```

**关键点**:
- 基于 Redis 的滑动窗口（使用时间窗口取整实现）
- 使用 `INCR` + `EXPIRE` 原子操作保证并发安全
- 不同的 `RateLimiter` 实例可用于不同场景（全局/登录/API）
- `decode_responses=True` 让 Redis 返回字符串而非字节序列
- 速率限制应在客户端 IP 而非用户级别，避免认证前的攻击

### 5.6 Redis 缓存（Caching）

**问题**: 频繁查询相同数据浪费数据库资源。Redis 缓存可以缓存不常变化的查询结果，显著降低数据库负载。

**代码示例:**

```python
# caching.py
import json
from collections.abc import Callable
from datetime import timedelta
from functools import wraps
from typing import Any

import redis.asyncio as aioredis

from app.config import get_settings

settings = get_settings()
redis_client = aioredis.from_url(settings.REDIS_URL, decode_responses=True)


def make_cache_key(prefix: str, *args, **kwargs) -> str:
    parts = [prefix]
    parts.extend(str(a) for a in args)
    parts.extend(f"{k}={v}" for k, v in sorted(kwargs.items()))
    return ":".join(parts)


async def cache_get(key: str) -> Any | None:
    data = await redis_client.get(key)
    if data is None:
        return None
    try:
        return json.loads(data)
    except (json.JSONDecodeError, TypeError):
        return data


async def cache_set(key: str, value: Any, expire: int = 300) -> None:
    serialized = json.dumps(value, default=str, ensure_ascii=False)
    await redis_client.setex(key, expire, serialized)


async def cache_delete(pattern: str) -> None:
    cursor = 0
    while True:
        cursor, keys = await redis_client.scan(cursor=cursor, match=pattern, count=100)
        if keys:
            await redis_client.delete(*keys)
        if cursor == 0:
            break


def cached(prefix: str, expire: int = 300):
    def decorator(func: Callable) -> Callable:
        @wraps(func)
        async def wrapper(*args, **kwargs):
            cache_key = make_cache_key(prefix, *args, **kwargs)
            cached_data = await cache_get(cache_key)
            if cached_data is not None:
                return cached_data

            result = await func(*args, **kwargs)
            await cache_set(cache_key, result, expire=expire)
            return result

        return wrapper
    return decorator


# 使用示例
from fastapi import APIRouter, Depends
from sqlalchemy import select
from sqlalchemy.orm import joinedload
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.models.post import Post
from app.schemas.post import PostRead

cache_router = APIRouter(prefix="/api/cached", tags=["caching"])


@cache_router.get("/posts/{post_id}", response_model=PostRead)
@cached(prefix="post", expire=60)
async def get_cached_post(post_id: int, db: AsyncSession = Depends(get_db)):
    result = await db.execute(
        select(Post).options(joinedload(Post.author)).where(Post.id == post_id)
    )
    post = result.unique().scalar_one_or_none()
    if not post:
        from fastapi import HTTPException, status
        raise HTTPException(status_code=404, detail="文章不存在")
    return post


# 手动缓存操作示例
async def get_post_stats(db) -> dict:
    cache_key = "stats:posts"
    cached = await cache_get(cache_key)
    if cached is not None:
        return cached

    from sqlalchemy import func
    result = await db.execute(
        select(func.count(), func.avg(func.length(Post.content)))
    )
    count, avg_length = result.one()
    stats = {"total_posts": count, "avg_content_length": round(avg_length or 0, 1)}
    await cache_set(cache_key, stats, expire=600)
    return stats
```

**curl 示例:**

```bash
# 首次请求 - 从数据库加载并写入缓存
curl http://localhost:8000/api/cached/posts/1

# 后续请求（60 秒内）- 从缓存返回，无数据库查询
curl http://localhost:8000/api/cached/posts/1
```

**关键点**:
- 缓存仅在数据不常变化时有效。写入操作后应主动失效相关缓存
- `json.dumps(value, default=str)` 处理无法自动序列化的类型（如 datetime）
- 使用 `scan` + `delete` 实现通配符批量清除缓存
- 缓存过期时间根据数据变化频率设定——评论列表缓存 30 秒，文章详情缓存 60 秒，统计数据缓存 600 秒
- 缓存键应包含查询参数，避免不同请求返回相同结果

### 5.7 游标分页（Cursor Pagination）

**问题**: 传统 `OFFSET/LIMIT` 分页在数据量大时性能急剧下降（OFFSET 越大的行扫描越多），且数据变化时可能导致重复或遗漏。游标分页基于索引扫描，性能恒定。

**代码示例:**

```python
# cursor_pagination.py
from datetime import datetime
from typing import Any

from fastapi import APIRouter, Depends, HTTPException, Query, status
from pydantic import BaseModel, ConfigDict
from sqlalchemy import select, desc
from sqlalchemy.orm import joinedload
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.models.post import Post
from app.schemas.post import PostRead
from app.core.dependencies import get_current_user, pagination_params


class CursorPage(BaseModel):
    items: list[PostRead]
    next_cursor: str | None
    has_more: bool
    page_size: int


def encode_cursor(value: Any) -> str:
    import base64, pickle
    return base64.urlsafe_b64encode(pickle.dumps(value)).decode()


def decode_cursor(cursor: str) -> Any:
    import base64, pickle
    try:
        return pickle.loads(base64.urlsafe_b64decode(cursor.encode()))
    except Exception:
        raise HTTPException(status_code=400, detail="无效的游标")


cursor_router = APIRouter(prefix="/api/cursor-posts", tags=["cursor-pagination"])


@cursor_router.get("", response_model=CursorPage)
async def list_posts_cursor(
    cursor: str | None = Query(None, description="分页游标（从上一页获取）"),
    page_size: int = Query(20, ge=1, le=100, description="每页数量"),
    db: AsyncSession = Depends(get_db),
):
    query = select(Post).options(joinedload(Post.author)).order_by(desc(Post.id))

    if cursor:
        cursor_value = decode_cursor(cursor)
        query = query.where(Post.id < cursor_value)

    query = query.limit(page_size + 1)
    result = await db.execute(query)
    posts = result.unique().scalars().all()

    has_more = len(posts) > page_size
    if has_more:
        posts = posts[:page_size]

    next_cursor = None
    if has_more and posts:
        last_post = posts[-1]
        next_cursor = encode_cursor(last_post.id)

    return CursorPage(
        items=list(posts),
        next_cursor=next_cursor,
        has_more=has_more,
        page_size=page_size,
    )


# 基于时间戳的游标分页
@cursor_router.get("/by-date", response_model=CursorPage)
async def list_posts_by_date(
    cursor: str | None = Query(None, description="基于创建时间的游标"),
    page_size: int = Query(20, ge=1, le=100),
    db: AsyncSession = Depends(get_db),
):
    query = select(Post).options(joinedload(Post.author)).order_by(desc(Post.created_at), desc(Post.id))

    if cursor:
        cursor_value = decode_cursor(cursor)
        query = query.where(
            (Post.created_at < cursor_value["created_at"]) |
            (
                (Post.created_at == cursor_value["created_at"]) &
                (Post.id < cursor_value["id"])
            )
        )

    query = query.limit(page_size + 1)
    result = await db.execute(query)
    posts = result.unique().scalars().all()

    has_more = len(posts) > page_size
    if has_more:
        posts = posts[:page_size]

    next_cursor = None
    if has_more and posts:
        last_post = posts[-1]
        next_cursor = encode_cursor({
            "created_at": last_post.created_at,
            "id": last_post.id,
        })

    return CursorPage(
        items=list(posts),
        next_cursor=next_cursor,
        has_more=has_more,
        page_size=page_size,
    )
```

```bash

# 第二页（使用上一页返回的 next_cursor）
```

**关键点**:
- 游标分页基于索引（如 `Post.id`）过滤，每页查询性能恒定
- `LIMIT page_size + 1` 技巧：多查一条判断是否有更多数据
- 基于 `id` 的游标最简单，但基于 `created_at` + `id` 复合游标更精确（支持按时间排序）
- `pickle` + `base64` 编码游标，也可使用更轻量的方式（如 `id` 直接 base64 编码）
- 游标分页不支持跳转到指定页码（如"第 5 页"），但大数据集场景下性能远超 offset 分页

### 5.8 事务性批量操作（Transactional Batch Operations）

**问题**: 批量操作中部分成功部分失败会导致数据不一致。事务确保所有操作要么全部成功，要么全部回滚。

**代码示例:**

```python
# transactional_batch.py
from fastapi import APIRouter, Depends, HTTPException, status
from pydantic import BaseModel, field_validator
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.models.post import Post
from app.models.comment import Comment
from app.models.user import User
from app.core.dependencies import get_current_user

tx_batch_router = APIRouter(prefix="/api/tx-batch", tags=["transactional-batch"])


class PostWithComments(BaseModel):
    post_title: str
    post_content: str
    comment_contents: list[str]

    @field_validator("comment_contents")
    @classmethod
    def validate_comments(cls, v: list[str]) -> list[str]:
        if len(v) > 10:
            raise ValueError("每篇文章最多 10 条评论")
        return v


class TxBatchCreateRequest(BaseModel):
    items: list[PostWithComments]

    @field_validator("items")
    @classmethod
    def validate_items(cls, v: list[PostWithComments]) -> list[PostWithComments]:
        if len(v) > 50:
            raise ValueError("单次批量创建不可超过 50 组")
        return v


class TxBatchResult(BaseModel):
    created_posts: int
    created_comments: int
    post_ids: list[int]


@tx_batch_router.post("/posts-with-comments", response_model=TxBatchResult, status_code=201)
async def batch_create_posts_with_comments(
    request: TxBatchCreateRequest,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    post_ids = []
    total_comments = 0

    for item in request.items:
        post = Post(
            title=item.post_title,
            content=item.post_content,
            user_id=current_user.id,
        )
        db.add(post)
        await db.flush()

        comments = [
            Comment(content=content, post_id=post.id, user_id=current_user.id)
            for content in item.comment_contents
        ]
        db.add_all(comments)

        post_ids.append(post.id)
        total_comments += len(comments)

    await db.commit()

    return TxBatchResult(
        created_posts=len(request.items),
        created_comments=total_comments,
        post_ids=post_ids,
    )


class MovePostsRequest(BaseModel):
    source_user_id: int
    target_user_id: int
    post_ids: list[int]


@tx_batch_router.post("/move-posts", status_code=200)
async def move_posts_to_user(
    request: MovePostsRequest,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    if current_user.id != request.source_user_id:
        raise HTTPException(status_code=403, detail="只能转移自己的文章")

    result = await db.execute(
        select(Post).where(
            Post.id.in_(request.post_ids),
            Post.user_id == request.source_user_id,
        )
    )
    posts = result.scalars().all()

    if len(posts) != len(request.post_ids):
        not_found = set(request.post_ids) - {p.id for p in posts}
        raise HTTPException(status_code=404, detail=f"以下文章不存在或无权操作: {not_found}")

    target_result = await db.execute(
        select(User).where(User.id == request.target_user_id)
    )
    target_user = target_result.scalar_one_or_none()
    if not target_user:
        raise HTTPException(status_code=404, detail="目标用户不存在")

    for post in posts:
        post.user_id = request.target_user_id

    await db.commit()

    return {
        "message": f"已转移 {len(posts)} 篇文章到用户 {target_user.username}",
        "moved": len(posts),
    }
```

**curl 示例:**

```bash
# 批量创建文章+评论（事务性）
curl -X POST http://localhost:8000/api/tx-batch/posts-with-comments \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "items": [
      {
        "post_title": "FastAPI 入门",
        "post_content": "FastAPI 是一个现代异步 Web 框架...",
        "comment_contents": ["好文章！", "学到了", "期待续集"]
      },
      {
        "post_title": "Docker 实践",
        "post_content": "Docker 容器化部署指南...",
        "comment_contents": ["非常有帮助", "已收藏"]
      }
    ]
  }'

# 转移文章所有权
curl -X POST http://localhost:8000/api/tx-batch/move-posts \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "source_user_id": 1,
    "target_user_id": 2,
    "post_ids": [1, 2, 3]
  }'
```

**关键点**:
- `db.flush()` 将 SQL 发送到数据库但不提交，获取自增 ID 后继续添加关联数据
- `await db.commit()` 在最后一次性提交，确保原子性
- `field_validator` 在 Pydantic v2 中替代 v1 的 `validator`，`@classmethod` 是必须的
- 事务内任何异常都会导致 `rollback()`，数据一致性由数据库保证
- 复杂业务逻辑在事务内完成后再统一提交，避免部分更新

```bash
#!/bin/bash
# scripts/verify.sh - API 健康检查与端到端验证

set -e

echo "=== 验证完成 ==="
```
